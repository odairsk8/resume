Professional Profile

It is hard to describe an entire career in a few words. I would be glad to talk about what I have achieved, opening positions and how I might fit into a development opportunity. Following a briefly describing of my career:

I am a passionate developer, since my father brought up our first computer, and I was fascinated by it. After that growing up, I always knew that I would work with computers. Later on, I chose to study a technical high school, where I discovered that I would like to become a programmer.

Jumping to my career, I have started in a Microsoft internship program in 2005, when I was studying at Universty, where they did not pay salary, offering free courses and training though. I am grateful to have had this life opportunity which opened doors and permitted myself to entry in the market.

Competition is tough in Brazil, you need to provide more than one service. There is where I have learned how to act as an analyst, developer, database admin and sometimes designer. Even though, later on, I chose to act more as a developer, aiming to specialisation as a C# Asp.Net developer. Furthermore, learning to work in more than one role allowed me to develop my communication skills, which is one of my competitive advantages.

Nowadays, my mindset is to add value to businesses throughout software development. My former experience has taught me how to deliver quality products within the established deadline, aiming to quality, maintenance and customization.

I am ready to join and work with a team straight away. The acknowledgement from old to cutting-edge technologies helps me to collaborate with a team quickly.

My tools

Coding

Writing clean code delivering software is one of my key skills. Throughout my journey as a software engineer, I have learned the importance of writing code that is easy to maintain. TDD, DDD, DRY, KISS, SOLID, DI, IOC, among others, are acronyms that are extremely important for me. My aim is always to try to apply well-known concepts when creating software in a way which is not going to delay the software delivering. Maintenance is also important, so it is important for me to write code that is easy to change.

AspNet. MVC

AspNet MVC is powerful, and if you know how to use its features it is possible to fastly deliver projects. I have written from small to large systems using MVC.Therefore, I am able to work from collaborating to plan a project from scratch to join a team without spending too much time to learn how the project works.

Business Analisys

Communication is key when developing software. I really believe that the programmer who knows how to efficiently communicate, extracting the maximum amount of information from stakeholders, and knows how to transform ideas into working software has a huge competitive advantage. Accordingly, this is the way I aim to work, applying my technical skills to add value to the business.

Database

As a developer you need to deal with Database, that is a fact. How it works, how to extract the maximum performance. Normalization, cardinality, indexes, joins and other words need to be within your vocabulary. My database experience is about modelling, accessing, reporting and daily data maintenance. I have worked with a large range of RDBMS such as SQL Server, Oracle and MySQL. Moreover, I am able to write data layers using ADO.Net, Entity Framework or to use the provided data access tool.

Front End

Front-end matters! This is how you present your product to customers, whether they are the company or general customer, they get impressed by the front end. My main experience is working as an Asp.Net developer, therefore, I am used to writing pages using javascript and HTML, as well as the razor syntax. I am able to create web pages using vanilla Javascript, Jquery, Bootstrap, Angular 2+. I would rather work with a web designer providing layouts, my experience with this kind of partnership tells that the results might be great.

Delivering products

After working with a bunch of methodologies, my favourite one is Agile. Timing might mean the difference between a successful and unsuccessful product. Delivering high quality, testable software in the right time window is a competitive advantage of a development team. I believe that agile helps with that challenge, bringing up a measurable system to the business.
